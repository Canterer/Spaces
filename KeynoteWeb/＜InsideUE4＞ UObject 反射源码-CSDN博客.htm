 UHT分析得到足够的类型元数据信息，利用该信息在程序内存中构建类型系统结构，这个过程称为注册。

一般程序的构建流程需要经过预处理、编译、汇编、链接。

UE在内存中模拟构建的过程，在概念上划分阶段：生成、收集、注册、链接。

UHT代码生成

在C++程序中的预处理是用来对源代码进行宏展开、预编译指令处理、注释删除等操作。

在引擎中创建空C++项目命名为Hello、创建一个不继承的MyClass类。编译后HUT会生成四个文件（位于
Hello\Intermediate\Build\Win64\Hello\Inc\Hello）
    HelloClasses.h：目前无用
    MyClass.generated.h:    MyClass的生成头文件
    Hello.generated.cpp:    该项目的实现编译单元
    Hello.generated.dep.h:    Hello.generated.cpp的依赖头文件，仅顺序包含MyClass.h而已

引擎默认生成的文件样式：

CURRENT_FILE_ID定义在MyClass.generated.h中
#define CURRENT_FILE_ID Hello_Source_Hello_MyClass_h，这是UHT通过分析文件得到的信息

如需要UMyClass(const FObjectInitializer& ObjectInitializer)的构造函数自定义实现，则定义GENERATED_UCLASS_BODY宏来生成。
#define DEFINE_DEFAULT_OBJECT_INITIALIZER_CONSTRUCTOR_CALL(TClass) \
	static void __DefaultConstructor(const FObjectInitializer& X) { new((EInternal*)X.GetObj())TClass(X); }

声明定义一个构造函数包装器。根据名字反射创建对象时，需要调用该类的构造函数。而类的构造函数不能用函数指针指向，因此用一个Static函数包装一下，变成一个"平凡"的函数指针，而且所有类的签名一致，就可以在UClass里用一个函数指针保存起来。
class COREUOBJECT_API UClass : public UStruct

{
    ...
        typedef void (*ClassConstructorType) (const FObjectInitializer&);
        ClassConstructorType ClassConstructor;
        ...
}

CallableFunc是C++里实现的，所以这里不需要定义函数体。而另外两个函数其实是在蓝图里定义的，就需要专门生成exec前缀的函数供蓝图虚拟机调用。

展开execCallableFunc后为：
void execCallableFunc( FFrame& Stack, void*const Z_Param__Result )  //蓝图虚拟机的使用的函数接口
{
    Stack.Code += !!Stack.Code; /* increment the code ptr unless it is null */
    { 
        FBlueprintEventTimer::FScopedNativeTimer ScopedNativeCallTimer;     //蓝图的计时统计
    	this->CallableFunc(); //调用我们自己的实现
    }
}
根据函数签名的不同会加上不同的参数传递。
