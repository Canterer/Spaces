视频数据通常由一系列图像描述，在文件中存储视频相对存储文件更具挑战性。"描述"视频的数据量比文本大几个数量级，因此需要对其进行优化和压缩才能使视频文件可用。视频通常包含一个或多个音轨，视频和音轨需要同步才能播放，其还包含字母和其他元数据。

视频文件两个基本要求：包含不同的多媒体元素、压缩这些多媒体元素以使其可用。

为了解决"包含不同的多媒体元素"要求，视频文件被定义为容器。容器有多种格式(MP4、MKV、MOV等)。一个视频为MP4，指其容器格式。

为了压缩文件，容器内的每个元素都以特定方式编码。常见的音、视频编解码器(encode/decode)有：H.264、H.265、AAC、MP3、FLAC。

Demuxing:打开容器并提取不同的流。Muxing：将流打包到容器文件中。Decoding：解码压缩流。Encoding:编码压缩流。将容器从一种格式更改为另一种格式的过程称为转换(Transmuxing)。将编码从一种格式更改为另一种格式的过程称为转码(Transcoding)。

    WorkerThread.ThreadStart(Electra::MakeDelegate(this, &FAdaptiveStreamingPlayerWorkerThread::WorkerThreadFN));

void FAdaptiveStreamPlayerWorkerThread::WorkerThreadFN()
{
    while(!bTerminate)
    {
        HaveWorkSignal.WaitTimeoutAndReset(1000 * 20);
        SingletonLock.Lock()
        for(int32 i=0;i<PlayerInstances.Num();++i
            PlayerInstances[i]->HandleOnce();
        SingletonLock.Unlock()
    }
}

FAdaptiveStreamingPlayer::InternalLoadManifest(const FString&URL, const FString& MimeType)
    FString mimeType = Playlist::GetMIMETypeForURL(URL);
    if(mimeType == Playlist::MIMETypeHLS)
        ManifestReader = IPlaylistReaderHLS::Create(this);//FPlaylistReaderHLS
    elseif(mimeType == Playlist::MIMETypeMP4)
        ManifestReader = IPlaylistReaderMP4::Create(this);//FPlaylistReaderMP4
    elseif(mimeType == Playlist::MIMETypeDASH)
        ManifestReader = IPlaylistReaderDASH::Create(this);//FPlaylistReaderDASH
    if(ManifestReader.IsValid())
    {
        DispatchEvent(FMetricEvent::ReportOpenSource(URL));
        ManifestReader->LoadAndParse(URL);
    }
