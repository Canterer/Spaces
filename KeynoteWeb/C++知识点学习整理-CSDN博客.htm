内存对齐是为了提高数据的读取效率。变量的内存地址需要是其长度的整数倍。

1.数据成员对齐规则：结构体（Struct）或联合体（Union）的数据成员，其第一个数据成员放在offset为0的地方，以后的每个数据成员要放在其大小的整数倍的offset处。

2.若结构体作为数据成员，即一个结构体中有一个结构体作为数据成员。该结构体成员以自身最长的数据成员大小作为自己的大小，遵循第一条规则。

3.结构体的总大小，必须是其内部数据成员中内存大小最长的整数倍。

总结：假设结构体（Struct）或联合体（Union）或基本数据类型都有一个内存对齐长度，每个类型其内存地址需要从其内存对齐长度的整数倍开始存放。基本数据成员的内存对齐长度是其类型所占大小，结构体或联合体的内存对齐长度取其数据成员的内存对齐长度最大值。（包含结构体的结构体也是按其规则判断计算）

函数的调用工作：

1.用实参初始化函数对应的形参。

2.将控制权转移给被调用函数。主调函数的执行被暂时中断，被调函数开始执行。

注意：实参与形参存在对应关系，但没有规定实参的求值顺序。编译器能以任意可行的顺序对实参求值。

C++中，名字有作用域，对象有生命周期。

名字的作用域是程序文本的一部分，名字在其中可见。

对象的生命周期是程序执行过程中该对象存在的一段时间。

我们用传递给函数的实参初始化形参对应的自动对象。对于局部变量对应的自动对象来说，则分为两种情况：如果变量定义本身含有初始值 ，就用这个初始值进行初始化；否则，如果变量定义本身不含初始值，执行默认初始化。这意味着内置类型的未初始化局部变量将产生未定义的值。

引用

对于函数形参不需要改变的，尽量使用常量引用。使用引用，对于一些具有较大的内存的实参就不用赋值给形参。而对一些类类型可能不支持系统赋值操作（深拷贝等）。使用引用作为形参会使函数对于实参的类型限制过于严格。引用类型的赋值只允许对应类型的实参（例如&int类型不会接受常数），而常引用可以被字面量初始化。

const分为底层const和高层const，当其作为形参时，实参赋值会忽视底层const。具有底层const的函数和没有const的函数实参赋值过程是一样的，所以会认为函数重定义报错。

const定义常量从汇编角度看，只给出了对应的内存地址。而#define定义的常量在内存中有若干个拷贝。

const 类型说明符 变量名  等同于  类型说明符 const 变量名

注意： * & 必须紧跟变量名

char* const p;//指针本身是常量

const char* p;//指针指向的内容是常量

const int func();// const 修饰函数返回值，返回值相对应的付给一个常量

C/C++ 内存类型

        C中，内存分为5个区：堆(malloc)、栈(如局部变量、函数参数)、程序代码区(存放二进制码)、全局/静态存储区(全局变量、static变量)和常量存储区。C++中有自由存储区(new)，自由存储区是C++通过new和delect动态分配和释放对象的抽象概念。通过重载操作符，可改用其他内存来实现自由存储。如全局变量做的对象池(placement new定位new)。C++编译器默认使用堆来实现自由存储。

        一个程序的内存布局是：栈在高地址向低地址扩散，堆在低地址向高地址扩散。两者共用一大块空闲内存区域。操作系统的角度，程序的地址空间是逻辑空间，实际运行时会转换成物理地址。即在程序的角度看，申请一个连续的内存空间，逻辑地址为连续的。但实际运行时，取地址和数据时，可能为不连续的内存物理地址。操作系统在段页式内存分配方式上，将程序分为主程序段、数据段。即程序逻辑上使用一大块连续逻辑地址作缓冲空间(无限大)，但实际上由于虚拟存储技术，只有实际用到该地址即存储数据，才会在外存中占用盘块。

        堆和栈的区别，栈为系统管理局部变量的结构，堆为用户申请、释放的内存空间。

        malloc/free，函数名用于申请一定大小内存，返回地址指针，需要类型转换。

        new/delect，操作符，调用时先分配内存，然后调用对象的构造函数，返回对象类型指针。

        new的过程中如果构造函数异常，系统负责对分配的内存触发delect。

        placement new定位new，用于在事先申请好的内存地址构建对象。

左值与右值引用、移动构造函数、万能引用、移动语义、完美转发：

        右值引用是移动语义、完美转发两个语言特性的底层语言机制。

        移动语义：针对临时变量、函数参数用来替换昂贵的复制操作。

        完美转发：可以接受任意实参的函数模板，井将其转发到其他函数，目标函数也会接受到与转发函数所接受的完全相同的实参。

        std::move 强制返回右值  std::forward 有条件返回右值
