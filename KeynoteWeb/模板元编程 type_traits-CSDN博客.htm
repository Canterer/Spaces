C++的类型推导(type deduction)。C++98时代只有一套类型推导规则即函数模板，C++11增加了auto和decltype，C++14扩展了auto和decltype的语境。让C++有了更高的适应性，改了某处类型而不必将相关内容全部改一遍，自动类型推导可以'传导'这种改动。

//模板类型推导
template<typename T>    //声明
void f(ParamType param) //参数类型
f(expr);                //调用

模板类型推导分三种情况：ParamType是指针或引用但非万能引用、ParamType是万能引用、ParamType即非指针且非引用。

//情况一：ParamType 非指针非引用
//param按值传递即为全新对象，是expr的副本。所以expr的const、volatile、引用类型都不会传递

若expr有引用类型，则忽略引用.若expr有const或volatile修饰，则一并忽略

//表现形式 
template<typename T>
void f(T param);//T param 可匹配任何expr类型
int x=11; f(x);            //T和param都是int
const int cx=x; f(cx);     //T和param都是int
const int& rx=x; f(rx);    //T和param都是int

const char* const ptr = "hello";
f(ptr);    //T和param都是const char*
//总结 T param 会丢失部分类型修饰。



//情况二：ParamType 是指针或引用
若expr是引用或指针类型，T的类型中会将引用/指针部分忽略
其后根据T的类型确定ParamType的类型

//表现形式 T& T*针对数组、函数两者表现不一
template<typename T>
void rf(T& param)//T& param仅可匹配 普通实参或引用类型实参
template<typename T>
void pf(T* param)//T* param仅可匹配 指针类型的expr

int x =11; rf(x);           //T是int，param是int&    匹配普通实参
const int cx = x; rf(x);    //T是const int, param是const int&    匹配普通实参
const int& rx = x; rf(x);   //T是const int, param是const int&    匹配普通实参


pf(&x);                      //T是int, param是int*
const int* px=&x; pf(px);    //T是const int, param是const int*
//总结 T&/T* 可匹配普通实参、引用或指针实参，当实参类型为引用或指针时，param等同实参类型。实参为普通实参时，T等同实参类型。 T或param 总是会存在一个等同于实参类型。


数组实参，按值传递给函数模板时会被推导为指针。C语言特性，数组退化成指针。
const char name[] = "hello";
template<typename T>
void f(T param)
f(name);    //T和param 均推导为指针 const char*
template<typeanem T>
void f(T& param)
f(name);    //T推导为数组 param 推导为数组引用 const char (&)[6]

函数类型也同样会退化成函数指针
void func(int)
template<typename T>
void f(T param)
f(func);    //T和param 均推导为函数指针 void (*)(int) 
template<typename T>
void f(T& param)
f(func);    //param 推导为函数引用 void (&)(int)



//情况三：ParamType 为万能引用
万能引用(Universal Reference),现也被称为转发引用(Forwarding Reference)。且形式限定很严格，固定为T&&。 const T&& 为右值引用。
template<typename T>
void f(T&& param)
若expr是左值，T和Param都会被推导成左值引用。Param与T类型相同
若expr是右值，T忽略引用部分，其后根据T的类型推导Param类型。Param等同于expr
int x = 11; f(x);         //expr为左值， T与Param 均推导为int&
const int cx=x; f(cx);    //expr为左值， T与Param 均推导为const int&
const int& rx=x; f(rx);   //expr为左值， T与Param 均推导为const int&
f(11);                    //expr为右值，T推导为int, Param推导为int&&

type_traits 用于萃取类型特征。注意部分类型特征需要编译器内建支持。std::is_standard_layout需要用类似__is_standard_layout()的编译器扩展intrinsic实现。
