Protocol Buffer是Google提出的一种高效的数据序列化协议。protoc.exe是ProtoBuf编译器，对其他语言的代码生成，就是安装其语言对应的插件。protoc进程降proto文件的信息封装为CodeGeneratorRequest传递给插件子进程，插件子进程将根据CodeGeneratorRequest中的信息，将生成的代码数据封装为CodeGeneratorResponse对象传递给protoc进程。插件进程从标准输入读取出CodeGeneratorRequest数据，将CodeGeneratorResponse数据写到标准输出。执行protoc --test_out =. test.proto将根据--{name}_out调用名为protoc-gen-{name}插件即protoc-gen-test。

如何快速编写一个protoc插件
https://www.cnblogs.com/smarticen/p/16940135.html

ProtoBuf针对Lua的插件：protoc-gen-lua、lua-protobuf

protoc-gen-lua：用法与C++版本比较接近，通过工具生成每个message的代码，在Lua里面new一个message进行赋值，最后通过msg:SerializeToString()将其序列化成二进制。

lua-protobuf：代码轻量、接口和pbc类似，自己解析proto协议文件，传Lua的table可以直接encode成二进制数据，decode二进制数据可以直接生成一个table。

pbc是云风大神早期的一个对protobuf的解析库，(git://github.com/cloudwu/pbc.git)

protobuf地址：Releases · protocolbuffers/protobuf · GitHub 寻找protoc-xx.x-win64.zip下载

LuaRocks是lua脚本语言的包管理工具，同python的pip、javascript的npm一样。地址：http://luarocks.github.io/luarocks/releases/ 下载对应的luarocks-x.xx.x-windows-64.zip

Lua 官网地址：The Programming Language Lua  Lua x.x.x released下载源码但需要编译。Lua Binaries Download 下载编译好的lua-x.x.x_Win64_bin.zip、lua-x.x.x_Win64_dllw6_lib.zip

./luarocks.exe install lua-protobuf    //前提MinGW安装好，全称Minimalist GNU For Windows，是个精简的Windows平台C/C++、ADA及Fortran编译器
./luarocks.exe --local config variables.LUA E:\XX\lua-5.4.2\
其中E:\XX\lua-5.4.2\ 其中存放着lua-x.x.x_Win64_bin与lua-x.x.x_Win64_dllw6_lib解压内容。
注意路径最后得接"\"
./luarocks.exe path --bin


安装VisualStudio会间接安装好MinGW编译器。
Lua的绑定实际上除了pb库以外还导出了四个库，而luarocks并不知道这一点。
所以如果你的luarocks是用VS编译的，则编译之后的pb.dll文件只会导出pb库。
虽然不影响使用，但是也比较不爽……这时候可以考虑下载代码手动编译
cl /O2 /LD /MT /DLUA_BUILD_AS_DLL /I/path/to/lua/include pb.c path/to/lua/lib
其中，O2指定最大化优化速度，LD表明生成dll文件，MT指定生成的dll不依赖运行时库。
声明LUA_BUILD_AS_DLL宏是为了导出pb.c里所有的符号，pb.c是唯一一个需要编译的.c文件，
而Lua提供的头文件和导入库需要手动制定位置。
